# TMS Test Deduplication

## Когда использовать
- Пользователь даёт xlsx-выгрузку из TestIT и просит найти дубли.
- Цель — отчёт по группам смысловых дубликатов с оценкой severity.

## Входные данные
xlsx-файл со структурой:
- колонка A — ID теста
- колонка C — название теста
- колонка F — шаг
- колонка H — ожидаемый результат
- строка с заполненными A и C = заголовок нового теста; последующие строки с пустыми A/C — шаги текущего теста.

## Пайплайн

Запускай скрипты строго по порядку из директории скилла. Все скрипты принимают `--help`.

### 1. Извлечение
```bash
python scripts/extract.py <input.xlsx> --out tests.json
```
Результат: `tests.json` вида `[{id, title, steps:[{action, expected}]}]`.

### 2. Кластеризация кандидатов
```bash
python scripts/cluster.py tests.json --out clusters.json --threshold 0.75 --min-size 2
```
Результат: `clusters.json` — список групп по 2–20 тестов-кандидатов. Быстро, без LLM.

### 3. Верификация через LLM
```bash
python scripts/verify.py clusters.json tests.json --out clusters_verified.json --batch-tokens 100000
```
Скрипт формирует батчи кластеров под лимит токенов и выводит **готовые промпты** в stdout (по одному на батч). Для каждого промпта:
1. Скопируй текст промпта.
2. Отправь его текущей LLM (qwen-coder-next).
3. Получишь строгий JSON — верни его скрипту через stdin / файл.

Альтернатива: скрипт поддерживает режим `--api` (OpenAI-совместимый endpoint) — тогда он вызывает LLM сам.

### 4. Отчёт
```bash
python scripts/report.py clusters_verified.json tests.json --out report.md
```
Открой `report.md` и покажи пользователю summary.

## Важно
- Никогда не пытайся скормить все 2500 тестов в LLM одним промптом — не влезет и не нужно.
- Этапы 1–2 детерминированные, кэшируй результаты.
- Если пользователь хочет повысить recall (ловить более тонкие перефразы) — предложи переключить эмбеддер на `sbert` (см. `scripts/embedders/`).
- Severity: `full` = полный дубль, удалять смело; `partial` = пересечение, ручное ревью; `none` = ложное срабатывание кластеризатора.
